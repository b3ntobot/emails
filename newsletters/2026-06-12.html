🧑‍💻 Agent Manager runs multiple agents without the chaos
Agent Manager gives every agent its own git worktree, branch, and isolated copy of your repo. That means you can run “refactor auth,” “update docs,” and “fix CI” at the same time without agents trampling each other’s files.
The workflow lives right inside Kilo Code. Open the multi-agent sidebar, create a new worktree, pick your prompt and model, then track each agent on a kanban-style board. When a run finishes, review the diff, open the worktree in VS Code, and merge it like any other feature branch.

🤖 KiloClaw is still the best way to Claw
KiloClaw keeps getting easier to set up, connect, and trust. Recent upgrades include the built-in morning brief, a dedicated @kiloclaw.ai forwarding address, one-click ClawBytes installs, simpler Google Calendar OAuth, and Composio support for hundreds of additional tools.
The bigger idea is early value without hand-wiring everything. Your Claw can summarize your day, flag urgent messages, prep around calendar conflicts, remember useful context, and import an existing OpenClaw identity if you are moving from a self-hosted setup.

💳 Coding Plans are live in Kilo
You can now buy external token plans directly with the credits already sitting on your Kilo balance. Coding Plans work across Kilo Code products and the Kilo Gateway, giving you more flexibility without juggling separate provider setup flows.
We are launching with MiniMax M3. The plan includes a 1M context window, multimodal input, generous monthly usage, and enough room for concurrent agent work. Pair it with Kilo Pass and your balance stretches even further across pay-as-you-go models, BYOK providers, and future plans.
Read about Coding Plans → View plans →
🖥️ Kilo Console beta is live
Kilo Console is a local, browser-based UI for the Kilo CLI. It gives you one place to manage projects, git worktrees, sessions, agents, tools, permissions, provider settings, notifications, and themes without hand-editing JSON config files.
The beta is built for feedback. Install the release candidate package, start the console from your terminal, open a project, and launch CLI sessions directly from the worktree you want to use. Then tell us what feels sharp and what still needs smoothing out.
