🚀 Kilo Code v7.3.12 is out
The latest release adds model override dropdowns, signed-in voice transcription, faster inline subagent streaming, and better autocomplete provider options. It is a practical quality-of-life release: less config guessing, more responsive agent sessions, and a smoother review surface.
More control over models. You can now choose agent model and variant overrides from dropdowns, and configure default task subagent model and reasoning effort while safely inheriting the calling agent model when an override is unavailable.
More polish in the flow. Voice transcription now appears automatically for signed-in Kilo provider users, streamed Markdown code blocks stay stable while output is still arriving, and readable diff highlights are restored across dark and light VS Code themes.

💸 Your keys, your provider bill, 0% Kilo markup
BYOK gives you direct provider billing plus a second source of truth inside Kilo Gateway. Import pay-as-you-go API keys or supported coding subscriptions, then use those models across the Kilo suite.
The dashboard is the useful part. Kilo Gateway shows total cost, request count, average cost per request, input/output tokens, spend by Kilo surface, spend by model, project breakdowns, trends over time, and CSV export for deeper analysis.
Supported providers include: Anthropic, AWS Bedrock, Chutes, DeepSeek, Fireworks, Google AI Studio, Inception, MiniMax, Mistral AI, Moonshot AI, Novita, Ollama Cloud, OpenAI, Perplexity, xAI, Xiaomi, and Z.ai. Coding-focused plans include BytePlus Coding Plan, Kimi Code, Mistral Codestral, Neuralwatt, Xiaomi Token Plans, and Z.ai Coding Plan.

👀 Human-in-the-loop keeps getting better
The v7 rewrite moved fast, and the feedback was clear: developers wanted more control back. The latest human-in-the-loop work is about making agent behavior easier to inspect before you approve it.
Visibility is the theme. Reasoning blocks stay expanded by default, terminal command blocks can collapse to a header, context usage is visible in the task header, patch operations show diffs in permission prompts, and the unified Changes panel brings workspace and session changes into one place.
