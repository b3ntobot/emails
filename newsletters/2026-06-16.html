🔁 Model freedom is the production feature
The best model is the one your workflow can actually rely on. When a frontier model disappears for policy, vendor, pricing, or availability reasons, your work should not disappear with it.
The alternatives are getting very real. GPT-5.5 continues to lead KiloBench, NVIDIA Nemotron 3 Ultra is currently free to use in Kilo, MiniMax M3 is built around strong cost efficiency, and Kimi K2.7 Code brings open weights plus serious MCP tool-use results.
In Kilo, switching is the point: use 500+ models through Kilo Gateway, bring your own keys when needed, and match the model to the task instead of rebuilding your workflow around one vendor.

🚀 SpaceX, Cursor, and the new AI coding economics
The reported SpaceX-Cursor deal is a compute-and-distribution story. Cursor has developer distribution; SpaceX has massive compute ambitions. Together, they show where AI coding strategy is moving.
The practical takeaway is simple. If the harness steers model demand, developers and teams need tools that preserve choice. Kilo’s model-agnostic approach means you can use the best fit today and switch tomorrow when the market changes again.

🛠️ v7.3.46 sharpens subagents, providers, and safety rails
The latest Kilo Code release keeps refining the agent loop. Task subagents now support model-specific reasoning overrides, including custom subagents with their own model and variant settings.
Also in the release: custom provider reasoning can be removed after it has been enabled, permission prompts and auto-approval rules are clearer, Redo restores reverted sessions correctly, question responses route back to the right worktree, and skill removal avoids recursively deleting working directories.
